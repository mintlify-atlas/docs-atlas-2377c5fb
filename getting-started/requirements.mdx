Before deploying a Kubernetes cluster with Kubespray, review the hardware, operating system, software, and network requirements below. Meeting all of these prerequisites before starting will prevent the most common deployment failures.

## Kubernetes version

Kubespray requires **Kubernetes v1.30 or later**. Older Kubernetes versions are not supported.

## Hardware requirements

These are the minimum memory values enforced by Kubespray. Your actual requirements will be higher depending on the workloads you intend to run. See the Kubernetes [Building Large Clusters](https://kubernetes.io/docs/setup/cluster-large/#size-of-master-and-master-components) guide for workload-based sizing.

| Node role | Minimum memory |
|---|---|
| Control plane | 2 GB |
| Worker node | 1 GB |

## Software requirements

### Ansible and Python

The machine that runs Ansible (your control node) needs:

| Package | Minimum version | Notes |
|---|---|---|
| Ansible | 11.x (pinned in `requirements.txt`) | See compatibility table below |
| Python | 3.11–3.13 | Required for Ansible >=2.18 |
| Jinja2 | 2.11+ | Templating engine used by Ansible |
| python-netaddr | Any | Required for IP address manipulation |
| cryptography | 47.0.0+ | Required for `community.crypto` module |
| jmespath | 1.1.0+ | Required for `json_query` Jinja2 filters |

The `requirements.txt` in the Kubespray repository pins exact versions:

```text
ansible==11.13.0
cryptography==47.0.0
jmespath==1.1.0
netaddr==1.3.0
```

Install them into a virtual environment to avoid version conflicts:

```bash
python3 -m venv kubespray-venv
source kubespray-venv/bin/activate
pip install -r requirements.txt
```

### Ansible Python compatibility

Choose an Ansible version that is compatible with the Python version available on your control node:

| Ansible version | Python version |
|---|---|
| &#62;=2.18.0, &#60;2.19.0 | 3.11–3.13 |

<Note>
  If pip reports that the latest supported Ansible version is older than what Kubespray requires, your Python version is likely too old. Python 3.9 is the minimum for Ansible 7.x; Python 3.11 is required for Ansible 2.18+.
</Note>

## Network requirements

### Target node connectivity

- **Internet access**: Target nodes must be able to pull container images from the internet. If your environment is air-gapped, see the [offline environment guide](/operations/offline-environment).
- **IPv4 forwarding**: All target nodes must be configured to allow IPv4 forwarding.
- **IPv6 forwarding**: Required only if you are using IPv6 for pods and services.

### Firewall

Kubespray does not manage firewall rules. You are responsible for configuring or disabling your firewall before deployment. An improperly configured firewall is the most common cause of node communication failures during deployment.

At minimum, the following must be allowed between all cluster nodes:

- TCP and UDP (all ports) for inter-node communication
- ICMP for health checking
- TCP port 6443 for the Kubernetes API server
- UDP for vxlan overlay traffic (required for Calico and Flannel CNI)

### Privilege escalation

If you run Kubespray from a non-root user account, the target servers must have a working privilege escalation mechanism (such as `sudo`). Pass `--become` or `-b` to `ansible-playbook`, or set `ansible_become: true` in your inventory.
