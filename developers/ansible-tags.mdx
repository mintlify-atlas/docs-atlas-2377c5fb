title: "Ansible tags reference for partial Kubespray playbook runs"
description: "Reference every Ansible tag in Kubespray playbooks and learn how to use --tags and --skip-tags to target specific installation phases."
---

Kubespray's playbooks are structured so that every logical step carries one or more Ansible tags. This lets you target a specific part of the cluster lifecycle — for example, reconfiguring DNS, reinstalling a CNI plugin, or downloading container images — without running the entire playbook. Understanding the tag taxonomy helps you iterate faster during development and perform surgical changes in production.

<Warning>
  Use `--tags` and `--skip-tags` only when you are certain which tasks will be affected. Skipping prerequisite tasks can leave the cluster in an inconsistent state.
</Warning>

## Using tags

Pass `--tags` to run only the tasks that carry those tags. Pass `--skip-tags` to exclude tasks carrying those tags. Both flags accept a comma-separated list.

```bash
ansible-playbook -i inventory/sample/hosts.ini cluster.yml --tags TAG1,TAG2
ansible-playbook -i inventory/sample/hosts.ini cluster.yml --skip-tags TAG1,TAG2
```

Combining both flags lets you narrow the scope precisely:

```bash
ansible-playbook -i inventory/sample/hosts.ini cluster.yml \
    --tags preinstall,facts \
    --skip-tags download,bootstrap_os
```

## Complete tag reference

|---|---|

## Practical examples

### Reconfigure DNS only

Apply only DNS configuration tasks, skipping OS bootstrap and image downloads:

```bash
ansible-playbook -i inventory/sample/hosts.ini cluster.yml \
    --tags preinstall,facts \
    --skip-tags download,bootstrap_os
```

### Remove the cluster DNS resolver from /etc/resolv.conf

```bash
ansible-playbook -i inventory/sample/hosts.ini \
    -e dns_mode='none' cluster.yml --tags resolvconf
```

### Pre-pull all container images locally

Download and cache all images on the Ansible controller without installing anything on cluster nodes:

```bash
ansible-playbook -i inventory/sample/hosts.ini cluster.yml \
    -e download_run_once=true \
    -e download_localhost=true \
    --tags download \
    --skip-tags upload,upgrade
```

### Reinstall a specific CNI plugin

```bash
ansible-playbook -i inventory/sample/hosts.ini cluster.yml \
    --tags calico,network,facts
```

### Upgrade only the etcd cluster

```bash
ansible-playbook -i inventory/sample/hosts.ini upgrade-cluster.yml \
    --tags etcd,upgrade
```
