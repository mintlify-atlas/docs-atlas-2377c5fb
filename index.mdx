Kubespray is a collection of Ansible playbooks that automates the deployment and lifecycle management of production-grade Kubernetes clusters. Whether you're deploying on AWS, Azure, GCE, OpenStack, vSphere, or bare metal, Kubespray handles provisioning, configuration, and ongoing operations with a single, composable toolset.

## Get started

<Steps>
  <Step title="Install Ansible">
    Install Ansible 2.14+ and Python dependencies on your control machine.

    ```bash
    pip install ansible netaddr
    ```
  </Step>
  <Step title="Clone Kubespray and prepare inventory">
    ```bash
    git clone https://github.com/kubernetes-sigs/kubespray.git
    cd kubespray
    cp -rfp inventory/sample inventory/mycluster
    # Edit inventory/mycluster/inventory.ini with your node IPs
    ```
  </Step>
  <Step title="Deploy your cluster">
    ```bash
    ansible-playbook -i inventory/mycluster/inventory.ini cluster.yml -b -v \
      --private-key=~/.ssh/id_rsa
    ```
  </Step>
  <Step title="Access your cluster">
    ```bash
    # kubeconfig is written to inventory/mycluster/artifacts/admin.conf
    export KUBECONFIG=inventory/mycluster/artifacts/admin.conf
    kubectl get nodes
    ```
  </Step>
</Steps>

<Note>
  Kubespray requires Ansible v2.14+, Jinja 2.11+, and python-netaddr on the machine running playbooks. Target nodes must have internet access unless deploying in [air-gap mode](/operations/offline-environment).
</Note>
